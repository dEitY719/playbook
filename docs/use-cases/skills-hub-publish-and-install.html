// 03 · Happy Path

주요 시나리오

스킬 한 개가 작성되어 동료의 환경에 도착하기까지의 다섯 단계.

S1
Author

스킬 제작

Author가 Claude Code에서 사용할 스킬 /visualize를 작성한다.

이 스킬은 임의의 콘텐츠(Markdown 파일, 대화 내용, URL, CSV/JSON 데이터, 코드 등)를 입력받아 하나의 자기완결형(single-file) HTML 파일로 변환한다. CSS·JS가 인라인되어 외부 의존 없이 어디서든 열리고 오프라인에서도 동작한다.

출력 형식은 슬라이드 덱, 인포그래픽, 대시보드, 차트, 플로우차트, 타임라인, 비교 테이블, 원페이저 등 다양하며, 라이트/다크 테마 토글·반응형 레이아웃·접근성(시맨틱 HTML, aria-label, prefers-reduced-motion)·인쇄/PNG 다운로드 메뉴를 기본으로 갖춘다.

S2
Consumer

공유 요청 발생

동료(Consumer)가 Author에게 /visualize 스킬을 공유해 달라고 요청한다.

S3
Publish

스킬 등록

Author는 Skills Hub에 /visualize 스킬을 등록한다.

  • 스킬 메타데이터(이름, 설명, 버전, 작성자, 태그)를 함께 제출한다.
  • 스킬 본문(파일/매니페스트)을 업로드한다.
S4
Install

스킬 탐색 및 설치

Consumer는 Skills Hub에서 /visualize 스킬을 검색하거나 공유 링크로 접근한다.

한두 번의 명령/클릭으로 스킬을 자신의 Claude Code 환경에 다운로드·설치한다.

S5
Discover

스킬 설명 페이지 제공

Skills Hub 웹 사이트는 각 스킬마다 가독성 좋은 상세 페이지를 제공한다.

  • 스킬 이름, 한 줄 설명, 사용 예시
  • 입력 / 출력
  • 설치 방법, 작성자, 버전 이력
// 04 · Postconditions

완료 상태

✓

Consumer의 Claude Code 환경에 /visualize 스킬이 정상 설치되어 호출 가능하다.

✓

Skills Hub에 해당 스킬이 검색 가능한 상태로 노출된다.

// 05 · Derived Requirements

핵심 요구사항

이 시나리오에서 자연스럽게 도출되는 시스템 책임.

REQ-01 · Publish

등록 채널

REQ-05 · Detail

스킬별 상세 페이지에서 README 렌더링, 사용 예시, 설치 가이드를 제공.

버전 관리

동일 스킬의 여러 버전을 관리하고 특정 버전 설치를 지원해야 한다.
