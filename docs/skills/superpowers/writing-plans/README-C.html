writing-plans

스펙을 구현 가능한 세부 실행계획으로 문서화하는 스킬

Problem

이 스킬이 해결하는 문제

구현 단계에서 파일/책임 경계가 흔들리는 문제를 줄인다

작업 단계가 커서 진행 상황과 검증 포인트가 불명확한 문제를 줄인다

Key Points

핵심 방법론

1

파일 구조와 책임 경계를 먼저 설계한다

2

2~5분 단위의 체크박스 작업으로 세분화한다

3

각 단계에 실제 코드/명령/예상 결과를 명시한다

4

계획 완료 후 실행 방식(서브에이전트/인라인)을 선택하게 한다

Team Guide

팀 활용 가이드

팀에서 이 스킬을 사용하는 규칙
  • PR/리뷰/디버깅/계획 수립 시 이 문서의 체크리스트를 먼저 확인한다.
  • 개인 선호보다 스킬의 게이트 조건(승인, 검증, 옵션 제시)을 우선 적용한다.
  • 예외가 필요하면 근거를 남기고 팀에 공유한다.

Example Usage

사용 예시

Prompt 
spec 기반으로 writing-plans 형식의 구현 계획서를 만들어줘. 단계별 코드/검증 명령까지 포함해.
