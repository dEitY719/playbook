01 · WHY

왜 Skill인가? — 한 마디로 "토큰"

처음 Skill을 써본 계기는 단순합니다. 토큰을 아끼고 싶어서. Anthropic 플랫폼의 진화를 따라가보면 의도가 명확합니다.

STAGE 01

MCP

Model Context Protocol — 외부 도구·데이터 연결
Context 비용
스키마가 통째로 로드 — 도구가 많을수록 무거움
STAGE 02

Agent

Sub-agents — 전문 역할 분리

★ STAGE 03 · WIN

Context 비용

02 · DAILY DRIVERS

내가 매일 쓰는 스킬 — "GitHub 파이프라인 4단"

자연어로도 다 됩니다. 그런데 Skill로 박아두면 얻는 게 3가지: 반복 입력 제로 · 품질 일관성 · 팀 공유. 아래 4개로 이슈 → 커밋 → PR → 리뷰 응대 한 사이클을 자동화합니다.

01

/gh:issue

대화를 GitHub Issue로
✘ Before · 자연어
지금 너와 대화한 내용을 요약해서 현재 repository의 issue로 등록해. label은 enhancement로 설정.
✔ After · Skill
/gh:issue
  • 현재 대화를 feature / bug / misc로 자동 분류해 구조화된 issue 본문 생성
  • gh issue create로 바로 등록 — 확인 프롬프트 없음
  • issue는 PR·블로그에서 재활용되므로 과도하게 압축하지 않음 (근거·결정 보존)
02

/gh:commit

이슈 자동 링크 커밋
✘ Before
지금 너와 작업한 내용을 커밋으로 만들어.
✔ After
/gh:commit
  • git status/diff/log를 병렬로 읽어 repo의 기존 커밋 스타일을 그대로 따라감
  • 최근 대화나 인자에서 #N을 찾아 Refs #N / Closes #N 자동 footer 추가
  • 항상 새 커밋 — amend 금지, hook 스킵 금지
03

/gh:pr

커밋 묶어 PR 생성
✘ Before
지금까지 작업한 커밋을 묶어서 PR 만들어. PR은 신규 브랜치에서 생성해.
✔ After
/gh:pr
  • base..HEAD 범위의 모든 커밋을 반영한 PR 본문 작성 (HEAD 하나만 보지 않음)
  • 필요 시 자동 push, 관련 이슈가 있으면 자동 링크
  • 결과는 PR URL 한 줄 — 바로 브라우저로
04

/gh:pr-reply

리뷰 코멘트 응대의 "예의"
✘ Before
동료가 PR#132에 리뷰를 달았어. 검토하고 타당하면 수용해. 리뷰 에티켓에 맞춰서 답글도 꼭 달아. 너는 답글 다는 것을 매번 놓치더라.
✔ After
/gh:pr-reply
  • 모든 리뷰 코멘트 (봇 포함 — gemini/sourcery/copilot)를 수집해 하나씩 판단
  • 수용 시 코드 수정까지, 거절 시 이유를 명확히 답글 — "조용한 무시"는 없음
  • 현재 브랜치 PR 자동 감지, /gh:pr-reply 123처럼 번호 지정도 가능
이 스킬은 "답글 다는 걸 매번 놓치더라"는 잔소리를 규칙으로 박아버린 사례. 자연어 지시의 휘발성을 Skill이 잡아주는 대표 예.
03 · FAVORITE

요즘 최애 — /visualize

/visualize some-doc.md ★ FAVORITE
  • 어떤 내용이든 단일 파일 HTML 시각화 생성 — 슬라이드 · 인포그래픽 · 대시보드 · 플로우차트 · 타임라인…
  • 라이트·다크 테마, 유틸 메뉴 (PNG 다운로드 · 인쇄), 시맨틱 HTML 등 평가 기준을 기본 충족
  • 생성 즉시 wslview/xdg-open/open으로 자동 실행, file:// URL까지 반환
왜 최애냐면 — README, 회의 노트, 제품 아이디어가 들어오면 5분 안에 공유 가능한 비주얼이 나옵니다. 동료한테 "이거 봐봐" 링크 하나 던지면 설명 끝. 말보다 이미지가 빠르니까요. (참고로 지금 보고 계신 이 페이지도 /visualize로 뽑은 겁니다.)
04 · CONCLUSION

결론 — "반복을 자산으로"

기준 자연어 Agent ★ Skill
Context 소모 매 지시마다 누적 상주 메타만 상주, 본문 온디맨드
재현성 그때그때 다름 에이전트 프롬프트에 의존 SKILL.md = 단일 소스
팀 공유 위키·복붙 설정 파일 필요 git 체크인하면 끝
수정 난이도 기억과 싸움 에이전트 튜닝 md 파일 편집
Skill의 본질은 "AI에게 내 업무 규칙을 영속화하는 것"입니다. 오늘 자연어로 세 번 이상 반복한 지시가 있다면 — 그건 이미 Skill 후보입니다.
