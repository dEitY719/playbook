# test-driven-development

## 한 줄 설명
Red-Green-Refactor 순서를 강제해 안정적으로 기능을 구현하는 TDD 스킬

## 이 스킬이 해결하는 문제
- 구현 후 테스트 방식으로 회귀 버그를 놓치는 문제를 줄인다.
- 요구사항을 코드로만 해석해 기대 동작이 흔들리는 문제를 줄인다.

## 핵심 포인트
- RED: 실패 테스트를 먼저 작성한다.
- GREEN: 테스트를 통과시키는 최소 구현만 추가한다.
- REFACTOR: 동작 유지 상태에서 구조를 정리한다.
- 각 단계는 실제 실행 결과로 검증한다.

## 팀 적용 가이드
- PR/리뷰/디버깅/계획 수립 시 이 문서의 체크리스트를 먼저 확인한다.
- 개인 선호보다 스킬의 게이트 조건(승인, 검증, 옵션 제시)을 우선 적용한다.
- 예외가 필요하면 근거를 남기고 팀에 공유한다.

## Example usage
```text
"이 기능을 test-driven-development 방식으로 진행해. 먼저 실패 테스트부터 작성해줘."
```
