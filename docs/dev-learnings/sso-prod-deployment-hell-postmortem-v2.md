# "로컬에선 됐는데요?" 사내망 SSO 배포 지옥에서 살아남기 (v2) 🛡️

> **"로컬에선 완벽했는데, 왜 운영 서버에만 올리면 안 될까?"**
> 삼성 DS 사내망(ADFS)에 SSO를 배포하며 겪은 2.5일간의 사투 기록입니다.
> Docker, Nginx, OIDC의 복합적인 빌런들을 때려잡은 실전 지침서입니다.

---

## 💡 핵심 통찰: SSO는 '기능'이 아니라 '파이프라인'이다

이번 장애 복구의 전환점은 에러를 보는 관점을 바꾼 것이었습니다. SSO는 아래 9단계 검문소를 순서대로 통과해야 하는 **파이프라인**입니다.

1. 로그인 클릭 → 2. IDP Redirect → **3. redirect_uri 검증** → **4. Callback 도달(포트/SSL)** → 5. id_token 수신 → 6. 서명 검증 → 7. Audience 검증 → **8. Issuer 검증** → 9. 로그인 완료

> **"에러 메시지가 바뀌는 건 새로운 버그가 아니라, 다음 단계로 진출했다는 승리의 신호입니다."**
> `SSL 에러(4단계)`를 해결하니 `mismatch(3단계)`가 보이고, 그걸 고치니 `Invalid issuer(8단계)`가 나타난 것입니다. 증상 변화를 '진전'으로 읽는 순간 디버깅이 차분해집니다.

---

## 3대 빌런 격파기

### 1. `ERR_SSL_PROTOCOL_ERROR` (Docker의 배신)
- **원인**: `--env-file .env.corp` 사용 시 기존 `.env`는 **보충이 아니라 완전히 대체**됩니다. 포트 변수가 증발하며 Nginx가 80번(HTTP)으로 떴고, HTTPS 핸드셰이크가 터졌습니다.
- **해결**: `.env.corp`를 `.env`의 내용을 모두 포함한 **'완전체'**로 관리하세요.

### 2. `redirect_uri_mismatch` (중첩 변수의 한계)
- **원인**: Docker Compose는 `${VAR:-${OTHER}}` 같은 **중첩 변수 확장을 지원하지 않습니다.** URL이 빈 값으로 평가되어 IDP 등록 정보와 어긋났습니다.
- **해결**: 환경변수 치환은 **단일 레벨**만 사용하고, 복잡한 값은 쉘에서 미리 계산하세요.

### 3. `Invalid issuer` (ADFS의 통수)
- **원인**: "로그인 URL에서 `/authorize`만 자르면 Issuer겠지?"라는 추론은 ADFS에서 통하지 않습니다. (실제: `/adfs/oauth2/authorize` vs `/adfs`)
- **해결**: Issuer는 URL이 아닌 **고유 식별자**입니다. 반드시 `discovery endpoint`에서 직접 복사해서 명시하세요.
