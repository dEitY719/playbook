# brainstorming

## 한 줄 설명
구현 전에 사용자 의도와 요구사항을 설계로 구체화하는 사전 브레인스토밍 스킬

## 이 스킬이 해결하는 문제
- 요구사항이 모호한 상태에서 바로 코딩해 재작업이 커지는 문제를 줄인다.
- 목적/제약/성공 기준을 먼저 합의해 팀 내 설계 해석 차이를 줄인다.

## 핵심 포인트
- 코드 작성 전에 반드시 설계 합의부터 진행한다.
- 질문은 한 번에 하나씩 던져 목적/제약/성공 기준을 명확히 한다.
- 2~3개 접근안을 비교하고 권장안을 제시한다.
- 설계를 문서화하고 사용자 리뷰 게이트를 통과한 뒤에만 `writing-plans`로 넘어간다.

## 팀 적용 가이드
- PR/리뷰/디버깅/계획 수립 시 이 문서의 체크리스트를 먼저 확인한다.
- 개인 선호보다 스킬의 게이트 조건(승인, 검증, 옵션 제시)을 우선 적용한다.
- 예외가 필요하면 근거를 남기고 팀에 공유한다.

## Example usage
```text
"새 결제 온보딩 플로우를 추가하고 싶어. brainstorming 절차로 요구사항 질문부터 시작해줘."
```
