Process Skills

Skill Usage Flow

워크플로우

Feature Development (풀 라이프사이클)
1 brainstorming 요구사항 탐색, 설계
2 writing-plans 스펙을 세부 태스크로 분해
3 executing-plans 태스크 실행 (단계별 TDD)
or subagent-driven-development 병렬, 같은 세션
5 requesting-code-review 리뷰어 서브에이전트 디스패치
6 receiving-code-review 피드백 평가 및 반영
7 finishing-a-development-branch 머지, PR, 정리
verification-before-completion은 모든 완료 선언 시점에서 실행됩니다. test-driven-development는 각 실행 단계 안에서 실행됩니다.
Bug Fix (짧은 사이클)
1 systematic-debugging 근본 원인 분석 우선
2 test-driven-development 버그에 대한 실패 테스트 작성
3 verification-before-completion 검증 증거 확인
4 finishing-a-development-branch 수정 사항 머지
병렬 태스크에는 dispatching-parallel-agents 또는 using-git-worktrees를 사용하세요.
