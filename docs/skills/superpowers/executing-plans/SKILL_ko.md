# 계획 실행하기

## 개요

계획을 불러오고, 비판적으로 검토한 뒤, 모든 작업을 실행하고, 완료되면 보고한다.

**시작 시 안내:** "executing-plans 스킬을 사용하여 이 계획을 구현하겠습니다."

**참고:** 인간 파트너에게 Superpowers는 서브에이전트에 접근할 수 있을 때 훨씬 더 잘 작동한다고 알려주세요. 서브에이전트를 지원하는 플랫폼(Claude Code 또는 Codex 등)에서 실행하면 작업 품질이 크게 향상됩니다. 서브에이전트를 사용할 수 있다면, 이 스킬 대신 superpowers:subagent-driven-development를 사용하세요.

## 프로세스

### 1단계: 계획 불러오기 및 검토
1. 계획 파일을 읽는다
2. 비판적으로 검토한다 - 계획에 대한 질문이나 우려 사항을 파악한다
3. 우려 사항이 있을 경우: 시작하기 전에 인간 파트너에게 제기한다
4. 우려 사항이 없을 경우: TodoWrite를 생성하고 진행한다

### 2단계: 작업 실행

각 작업에 대해:
1. in_progress로 표시한다
2. 각 단계를 정확히 따른다 (계획에는 세분화된 단계가 있다)
3. 명시된 대로 검증을 수행한다
4. completed로 표시한다

### 3단계: 개발 완료

모든 작업이 완료되고 검증된 후:
- 안내: "finishing-a-development-branch 스킬을 사용하여 이 작업을 마무리하겠습니다."
- **필수 하위 스킬:** superpowers:finishing-a-development-branch를 사용한다
- 해당 스킬을 따라 테스트를 검증하고, 선택지를 제시하고, 선택을 실행한다

## 멈추고 도움을 요청해야 할 때

**다음 상황에서는 즉시 실행을 중단한다:**
- 차단 요소에 부딪혔을 때 (누락된 의존성, 테스트 실패, 불명확한 지시)
- 계획에 시작을 방해하는 치명적인 결함이 있을 때
- 지시를 이해하지 못할 때
- 검증이 반복적으로 실패할 때

**추측하지 말고 명확한 확인을 요청한다.**

## 이전 단계로 돌아가야 할 때

**검토(1단계)로 돌아가는 경우:**
- 파트너가 피드백을 기반으로 계획을 업데이트했을 때
- 근본적인 접근 방식을 재고해야 할 때

**차단 요소를 억지로 돌파하지 않는다** - 멈추고 질문한다.

## 기억할 사항
- 먼저 계획을 비판적으로 검토한다
- 계획 단계를 정확히 따른다
- 검증을 건너뛰지 않는다
- 계획이 지시할 때 스킬을 참조한다
- 막혔을 때는 멈추고 추측하지 않는다
- 명시적인 사용자 동의 없이 main/master 브랜치에서 구현을 시작하지 않는다

## 통합

**필수 워크플로우 스킬:**
- **superpowers:using-git-worktrees** - 필수: 시작하기 전에 격리된 작업 공간을 설정한다
- **superpowers:writing-plans** - 이 스킬이 실행할 계획을 작성한다
- **superpowers:finishing-a-development-branch** - 모든 작업 후 개발을 완료한다
