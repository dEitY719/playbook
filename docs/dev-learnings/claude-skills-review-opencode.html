Workflow: Issue → Commit → PR

1. /gh:issue
지금 너와 대화한 내용을 요약해서 현재 repository의 issue로 등록해. → /gh:issue

feature / bug / misc 자동 분류 • gh issue create 바로 등록

2. /gh:commit
지금 너와 작업한 내용을 커밋으로 만들어. → /gh:commit

repo 스타일 자동 학습 • #N 이슈 자동 링크 • amend 금지

3. /gh:pr
지금까지 작업한 커밋을 묶어서 PR 만들어. → /gh:pr

모든 커밋 반영 • 자동 push • PR URL 반환

4. /gh:pr-reply
동료가 PR#132에 리뷰를 달았어. 검토하고 타당하면 수용해. → /gh:pr-reply

모든 코멘트 수용/거절 판단 • "조용한 무시" 방지

자연어 vs Agent vs Skill

기준 자연어 Agent Skill
Context 소모 매 지시마다 누적 상주 메타만 상주, 본문 온디맨드
재현성 그때그때 다름 에이전트 프롬프트에 의존 SKILL.md = 단일 소스
팀 공유 위키·복붙 설정 파일 필요 git에 체크인하면 끝
수정 난이도 기억과 싸움 에이전트 튜닝 md 파일 편집
핵심 메시지: AI에게 내 업무 규칙을 영속화하라.
자연어로 세 번 이상 반복한 지시가 있다면 — 그건 이미 Skill 후보입니다.

도구 중립: 환경별 대응

Claude를 못 쓰는 환경에서도 패턴 그대로 적용

이 글의 용어 OpenCode Roo Code
Skill (SKILL.md) Agent / Command (.opencode/) Mode (.roomodes 또는 Global Modes)
/gh:pr 같은 슬래시 호출 /agent <name> · /<command-name> Mode 전환 후 프롬프트
마켓플레이스 팀 레포에 .opencode/ 체크인 .roomodes 파일을 git 공유
Context 소모 특성 Agent 메타 상주, 본문은 호출 시 로드 Mode 정의는 경량, 선택 시 system prompt 교체
자연어 → Skill 파일로 영속화

설치 방법

Claude Code 세션 안에서 두 줄이면 끝:

/plugin marketplace add dEitY719/claude-skills /plugin install visualize

필요한 스킬만 골라 설치:

/plugin install gh:issue /plugin install gh:commit /plugin install gh:pr /plugin install gh:pr-reply

업데이트 / 제거:

/plugin update visualize /plugin uninstall visualize
