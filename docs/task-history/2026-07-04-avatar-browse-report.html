왜 큰 기능인가

DB 스키마 · 마이그레이션 AI 결정적 분류기 백그라운드 워커 가역 병합 거버넌스 i18n (ko/en) seed · 데모 데이터 E2E 재현 도구 브라우즈 UI

단일 UI 티켓이 아니라 정보구조 자체의 재설계였다. RFC와 동료 3인 리뷰(UX·백엔드/데이터·AI 거버넌스)를 거쳐 방향을 확정하고, 6개 Plan으로 분해해 리뷰 가능한 24개 PR 스택으로 병합했다. 유일한 비가역 작업인 hard-merge는 config OFF(soft-merge only)로 출시해 롤백 가능성을 보장했다. +2.6만 라인 · 367 파일 · 8개 계층을 관통하는 규모에서, 월–토 6일(주말 포함) 소요는 정상 범위다.
