# executing-plans

## 한 줄 설명
작성된 구현 계획을 단계별로 실행하고 체크포인트를 통과시키는 스킬

## 이 스킬이 해결하는 문제
- 계획 문서와 실제 구현이 어긋나는 실행 편차를 줄인다.
- 중간 검증 없이 진행해 후반에 결함이 몰리는 문제를 줄인다.

## 핵심 포인트
- 계획 문서를 먼저 비판적으로 검토하고 의문점을 해결한다.
- 작업별로 in_progress→검증→completed 순서로 진행한다.
- 모든 작업 완료 후 `finishing-a-development-branch`로 마무리한다.

## 팀 적용 가이드
- PR/리뷰/디버깅/계획 수립 시 이 문서의 체크리스트를 먼저 확인한다.
- 개인 선호보다 스킬의 게이트 조건(승인, 검증, 옵션 제시)을 우선 적용한다.
- 예외가 필요하면 근거를 남기고 팀에 공유한다.

## Example usage
```text
"이제 plan 문서대로 그대로 구현해줘. executing-plans 방식으로 체크포인트 포함해서 진행해."
```
