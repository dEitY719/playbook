# writing-skills

## 한 줄 설명
새 스킬 작성/개선을 TDD 방식으로 검증하며 완성하는 스킬

## 이 스킬이 해결하는 문제
- 스킬을 검증 없이 배포해 실제 적용성이 낮아지는 문제를 줄인다.
- 스킬 설명이 모호해 검색/트리거 정확도가 낮아지는 문제를 줄인다.

## 핵심 포인트
- 스킬 문서 작성을 RED-GREEN-REFACTOR로 운영한다.
- baseline 실패(스킬 없음)와 적용 후 성공(스킬 있음)을 비교 검증한다.
- 합리화 루프홀을 표/레드플래그로 명시적으로 봉쇄한다.
- CSO(검색 최적화) 관점으로 name/description/키워드를 설계한다.

## 팀 적용 가이드
- PR/리뷰/디버깅/계획 수립 시 이 문서의 체크리스트를 먼저 확인한다.
- 개인 선호보다 스킬의 게이트 조건(승인, 검증, 옵션 제시)을 우선 적용한다.
- 예외가 필요하면 근거를 남기고 팀에 공유한다.

## Example usage
```text
"새 스킬을 writing-skills 절차로 만들자. baseline 실패 시나리오부터 설계해줘."
```

## 산출물
- 한국어 가이드: `SKILL_ko.md`
- 원문: `superpowers/5.0.7/skills/writing-skills/SKILL.md`
## 추천 학습 순서
1. `SKILL_ko.md`를 먼저 읽어 전체 흐름을 파악한다.
2. 실제 적용 전에 원문 `SKILL.md`의 세부 규칙(금지/예외)을 재확인한다.
3. 작은 작업에서 1~2회 반복 적용 후 팀 표준으로 확장한다.
