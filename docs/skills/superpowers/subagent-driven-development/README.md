# subagent-driven-development

## 한 줄 설명
작업 단위를 서브에이전트로 위임하고 단계별 리뷰로 통제하는 개발 스킬

## 이 스킬이 해결하는 문제
- 큰 작업을 한 세션에서 처리하며 집중력이 분산되는 문제를 줄인다.
- 위임 결과를 무검증으로 합치는 품질 리스크를 줄인다.

## 핵심 포인트
- 작업을 독립적인 단위로 분해해 서브에이전트에 할당한다.
- 각 단위 완료 후 스펙 준수/코드 품질 2단계 리뷰를 수행한다.
- 필요 시 모델 선택을 조정하고 상태를 모니터링해 병목을 해소한다.

## 팀 적용 가이드
- PR/리뷰/디버깅/계획 수립 시 이 문서의 체크리스트를 먼저 확인한다.
- 개인 선호보다 스킬의 게이트 조건(승인, 검증, 옵션 제시)을 우선 적용한다.
- 예외가 필요하면 근거를 남기고 팀에 공유한다.

## Example usage
```text
"plan의 Task 1~4를 subagent-driven-development 방식으로 태스크별 위임해서 진행해줘."
```
