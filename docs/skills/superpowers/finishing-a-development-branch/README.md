# finishing-a-development-branch

## 한 줄 설명
개발 브랜치 완료 후 테스트 검증, 통합 방식 선택, 정리까지 마무리하는 스킬

## 이 스킬이 해결하는 문제
- 완료 시점에 머지/PR/보류/폐기 판단이 뒤섞이는 문제를 줄인다.
- 테스트 확인 없이 브랜치를 정리해 품질 리스크가 생기는 문제를 줄인다.

## 핵심 포인트
- 먼저 테스트 통과를 증거로 확인한다.
- base 브랜치를 확인한 뒤 4가지 선택지(로컬 머지/PR/유지/폐기)를 제시한다.
- 선택지에 맞는 git 절차를 수행하고 필요 시 worktree를 정리한다.

## 팀 적용 가이드
- PR/리뷰/디버깅/계획 수립 시 이 문서의 체크리스트를 먼저 확인한다.
- 개인 선호보다 스킬의 게이트 조건(승인, 검증, 옵션 제시)을 우선 적용한다.
- 예외가 필요하면 근거를 남기고 팀에 공유한다.

## Example usage
```text
"구현 완료했으니 finishing-a-development-branch 절차로 마무리 옵션을 제시해줘."
```

3. 작은 작업에서 1~2회 반복 적용 후 팀 표준으로 확장한다.
